API Documentation

This is a simple API implementation. Only main factors are taken into consideration as required. Edge cases are not taken into consideration such as providing wrong ID, Date or wrong URL etc to keep it simple. Implementation for edge cases can be done easily.

Users can just provide medallion id to get all the trips details for the given medallion. As well as, users can provide date parameter to pull data for the specified date.

By default (if no extra parameters are provided) the API will return output as TEXT and also all results will be cached. Users can ask for JSON output just by providing &output=json parameter.

Cache can be cleared by clicking the Clear Cache button (At top left corner of the page). Users can also ask for fresh data and ignore the cache by providing &cache=false parameter.

Some features and examples are bellow:

1. Simple Trip - Text

Add /api/v1/trips at the end of the main url then add ?medallion={MEDALLION ID}
Example: http://127.0.0.1:5000/api/v1/trips?medallion=A0B5AF0F9B31690CEBB51ECD27D2BE71

2. Simple Trip - Json

Add &output=json at the end of any url to get JSON as output.
Example: http://127.0.0.1:5000/api/v1/trips?medallion=A0B5AF0F9B31690CEBB51ECD27D2BE71&output=json

3. Trip with Date

Add &date={YYYY-MM-DD} at the end of any url to get trip data from the specified date.
Example: http://127.0.0.1:5000/api/v1/trips?medallion=A0B5AF0F9B31690CEBB51ECD27D2BE71&date=2013-12-06

4. Trips with Multiple Medallions

Add &medallion={MEDALLION ID} as much as you want after the first medallion id starts with (?) to get trip data from all the ids.
Example: http://127.0.0.1:5000/api/v1/trips?medallion=A0B5AF0F9B31690CEBB51ECD27D2BE71&medallion=8F0A1E787329C1C08F8F9120EE68E056&medallion=A21033DE83739A12D04B23384D1A3CC5

5. Force to Ignore The Cache and Ask for Fresh Data

Add &cache=false at the end of the url to ignore the cache and ask for fresh data.
Example: http://127.0.0.1:5000/api/v1/trips?medallion=A0B5AF0F9B31690CEBB51ECD27D2BE71&output=json&cache=false
